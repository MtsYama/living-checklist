# 示例 · 暑期欧洲 + 日本之行 · 原始输入(模拟语音转文字)

下面这段是用户**直接对 AI 口述**(语音录入自动转文字)的需求,一口气说下来、没分点 —— 这正是这套工具的用法演示:把一团乱麻的口述,理成有时序、可勾选、带 deadline 的清单。

> 7 月 15 到 8 月 15 我放假,想去欧洲转转,主要法国和意大利,然后还想顺路拐去趟日本,我中国护照。去程上海飞法国的机票我已经订好了,回程还没定。你帮我把这一趟理成一个能勾的清单 —— 签证该办哪些、什么时候办,申根的还有日本的;机票还有几个城市之间怎么走;住宿订哪几天。吃的我挺有讲究,意大利我想吃好一点、高档些的意餐;法国反倒不太爱那种 fine dining,更想吃法国乡下的农家菜;日本就想吃 omakase。有几家特别想去的餐厅得提前订位。博物馆,法国卢浮宫肯定要去,能免费当然最好,不过要是有哪天平时人特别少,我宁可挑那天去,不挤比省钱重要;其它城市有啥有名的特展也帮我看看,要不要提前买票、哪几天免费或者便宜点,我日期能调。对了还想带点礼物回来,买多了可能得寄。

---

**Persona 背景句**(给文章介绍背景用 · 不在 prompt 内):
> 我爱看展,出去玩也悠着来、不爱赶。

**场景前情提要**(放文章 / 演示开场白 · 不在 prompt 内 · 三条腿三个信息来源):
> - **武汉 → 上海**:助手在**本地 session** 里有这程的记录(含订单 / 截图)→ 它知道这程,也从记录里拿到我的**身份证号 → 姓名、生日**;但 **session 没有护照号**(国内段用不到)。
> - **上海 → 法国**:**我自己在那段话里说已经订好了**。
> - **回程**:我没提、也没订 → 它问我。
>
> 这次我只口述了上面那段需求,它就用这套 skill 生成了下面这份清单:**有据可查的(本地有记录 / 我口头说订的)预填好、标完成,不知道的(护照、签证细节、回程)留 placeholder 问我**。

**给读者的背景**(尤其持免签护照的读者 · 放文章 / README 框架,不在 prompt 内):中国护照对法、意(申根)和日本**都没有免签待遇**,必须提前办签证 —— 这正是清单里「签证」排头等大事的原因。持美 / 欧 / 日等免签护照的读者可能没这个困扰、容易低估它的分量。
